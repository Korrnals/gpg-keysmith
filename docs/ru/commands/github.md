# `github`

Загрузить GPG-публичный ключ в GitHub, выставить секреты Actions репозитория и открыть PR с файлом ключа.

## Синопсис

```
keysmith github [флаги]
```

`github` выполняет три действия:

1. Загружает публичный ключ в GPG keys аутентифицированного пользователя GitHub (REST API `users/gpg_keys`).
2. Сохраняет приватный ключ и парольную фразу как секреты Actions репозитория `GPG_PRIVATE_KEY` и `GPG_PASSPHRASE` (через `gh secret set`).
3. Коммитит файл публичного ключа в целевой репозиторий на ветку `chore/add-gpg-public-key` и открывает pull request.

Требуется GitHub PAT со scope `admin:gpg_key` (для загрузки публичного ключа) и `repo` + `admin:repo_hook` (для секретов репозитория). Для шага секретов должен быть установлен `gh` CLI — `gpg-keysmith` вызывает `gh secret set`, чтобы не тянуть зависимость от нативной привязки libsodium.

## Разрешение токена

Токен берётся только из переменных окружения, никогда из флага:

1. Переменная окружения, имя которой задано в `config.github.token_env` (по умолчанию `GITHUB_TOKEN`)
2. `GH_TOKEN` как запасной вариант

Флаг `--token` утекал бы через `ps` и `/proc/cmdline` — он удалён при усилении безопасности. Парольная фраза по той же причине использует stdin; два механизма симметричны.

## Флаги

| Флаг | Тип | По умолчанию | Описание |
|---|---|---|---|
| `--keyid` | string | `""` | GPG key id для экспорта (если пусто — выбрать интерактивно из `detect`) |
| `--pubkey-file` | string | `""` | Прочитать armored публичный ключ из этого файла вместо вызова `gpg --export` |
| `--repo` | string | `""` (обязательно) | Целевой репозиторий как `owner/name` |
| `-h`, `--help` | bool | `false` | Вывести помощь для `github` |
| `--config` | string | `~/.config/gpg-keysmith/config.yaml` | Путь к файлу конфигурации (глобальный флаг) |

`--repo` обязателен. Если `--keyid` пустой, ключ выбирается интерактивно из `gpg --list-secret-keys`. Если задан `--pubkey-file`, armored публичный ключ читается из файла вместо вызова `gpg --export`.

## Примеры

### Загрузить в репозиторий (интерактивный выбор ключа)

```bash
export GITHUB_TOKEN=ghp_ваш_pat_с_правами_repo_admin_gpg_key
keysmith github --repo owner/name
```

### Загрузить конкретный ключ неинтерактивно

```bash
keysmith github --repo owner/name --keyid F49BE957CD553B1C
```

### Использовать заранее экспортированный файл ключа (без `gpg --export`)

```bash
keysmith export --keyid F49BE957CD553B1C --pubkey ./gpg-public-key.asc
keysmith github --repo owner/name --pubkey-file ./gpg-public-key.asc
```

### Использовать другую переменную окружения для токена

```bash
# В ~/.config/gpg-keysmith/config.yaml:
#   github:
#     token_env: MY_GH_TOKEN
export MY_GH_TOKEN=ghp_...
keysmith github --repo owner/name
```

## Требуемые scope'ы PAT

| Scope | Зачем |
|---|---|
| `admin:gpg_key` | Загрузить публичный ключ в GPG keys пользователя |
| `repo` | Выставить секреты репозитория, закоммитить файл ключа, открыть PR |
| `admin:repo_hook` | Требуется вместе с `repo` для записи секретов репозитория |

## Замечания

- **Дедупликация публичного ключа.** `github` находит существующие GPG-ключи по fingerprint и не загружает повторно.
- **Секреты через `gh`.** Секреты репозитория используют шифрование libsodium sealed-secret, которое `gh secret set` выполняет нативно. Переизобретать это на Go означало бы тянуть нативную привязку libsodium; вызов `gh` сохраняет зависимостную поверхность маленькой, а логику шифрования — в проверенном инструменте.
- **Ветка PR.** Файл ключа коммитится в ветку `chore/add-gpg-public-key`, после чего открывается PR в ветку по умолчанию.
- **Валидация owner/repo.** `ValidateOwnerRepo` отвергает всё вне `^[A-Za-z0-9._-]+$` до того, как имя репозитория попадёт в путь URL; `url.PathEscape` применяется как дополнительная мера. См. [Безопасность](../security.md).
- **Runtime-зависимости.** Требуется `gh` (GitHub CLI) для шага секретов и `gpg`, если не задан `--pubkey-file`.
- **Обработка парольной фразы.** Парольная фраза подаётся в `gpg --export-secret-keys` через stdin при перехвате приватного ключа (только если не задан `--pubkey-file`). Она никогда не является аргументом CLI.
