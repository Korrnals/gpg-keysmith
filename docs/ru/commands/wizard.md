# `wizard`

Запустить полный поток настройки `gpg-keysmith` от начала до конца.

## Синопсис

```
keysmith wizard [флаги]
```

Мастер оркестрирует шесть этапов по порядку:

```mermaid
flowchart LR
  D[detect] --> G[generate]
  G --> E[export]
  E --> GC[git-config]
  GC --> GH[github]
  GH --> P[publish]
```

На каждом шаге запрашивается подтверждение, при сбое предлагается retry / skip / abort, а результат шага пишется в `~/.config/gpg-keysmith/state.json`, чтобы прерванный запуск возобновился с последнего успешного шага. При полном завершении файл состояния очищается.

Флаги предзаполняют промпты survey; любой пустой флаг запрашивается интерактивно. `--reset` очищает файл состояния и запускает заново.

## Флаги

| Флаг | Тип | По умолчанию | Описание |
|---|---|---|---|
| `--comment` | string | `""` | Необязательный комментарий для user id ключа (запрашивается, если пусто) |
| `--email` | string | `""` | Email для ключа + git `user.email` (запрашивается, если пусто) |
| `--expiry` | string | `"0"` | Спецификация срока действия (`0` = бессрочно, `2y` = 2 года) |
| `--key-length` | int | `4096` | Длина RSA-ключа в битах |
| `--keyserver` | string | `"all"` | Целевой сервер ключей: `all` (по умолчанию), `openpgp` или `ubuntu` |
| `--name` | string | `""` | Настоящее имя для ключа + git `user.name` (запрашивается, если пусто) |
| `--repo` | string | `""` | Целевой репозиторий GitHub как `owner/name` (запрашивается, если пусто) |
| `--reset` | bool | `false` | Очистить файл состояния и запустить заново (игнорировать прежний прогресс) |
| `--state-path` | string | `~/.config/gpg-keysmith/state.json` | Переопределить расположение файла состояния |
| `-h`, `--help` | bool | `false` | Вывести помощь для `wizard` |
| `--config` | string | `~/.config/gpg-keysmith/config.yaml` | Путь к файлу конфигурации (глобальный флаг) |

## Примеры

### Полностью интерактивно (запросить всё)

```bash
export GITHUB_TOKEN=ghp_ваш_pat_с_правами_repo_admin_gpg_key
keysmith wizard
```

Мастер проходит каждый шаг, запрашивая имя, email, комментарий, срок, длину ключа, репозиторий и парольную фразу по необходимости.

### Предзаполнить name, email, repo (остальное запрашивается)

```bash
keysmith wizard --name "Jane Doe" --email jane@example.com --repo owner/name
```

### Сгенерировать ключ на 2 года с комментарием

```bash
keysmith wizard --name "Jane Doe" --email jane@example.com --repo owner/name \
  --comment "acme-corp signing" --expiry 2y
```

### Опубликовать только на keys.openpgp.org

```bash
keysmith wizard --name "Jane Doe" --email jane@example.com --repo owner/name \
  --keyserver openpgp
```

### Возобновить после сбоя

```bash
# Предыдущий запуск упал на шаге github
keysmith wizard   # возобновляется со шага github; detect/generate/export/git-config пропускаются
```

### Начать заново и игнорировать прежний прогресс

```bash
keysmith wizard --reset
```

## Замечания по безопасности

- **Парольная фраза всегда маскированный промпт.** Парольная фраза собирается через `survey.Password` и подаётся в `gpg` через stdin. Она никогда не читается из флага (что утекало бы через историю shell / `ps`).
- **В файле состояния нет секретов.** Файл состояния хранит только имена шагов, key id, email и репозиторий. Поля `Passphrase`, `PrivateKey` и `PubKeyArmor` структуры `WizardState` несут тег `json:"-"` — они хранятся в памяти между шагами и уничтожаются в конце запуска. Инвариант проверяется тестом `TestSaveStateOmitsSecrets`. См. [Безопасность](../security.md).
- **Runtime-зависимости.** Требуется `gpg`, `git` и `gh` (для шага github), а также переменная окружения с GitHub PAT.

## Поведение по шагам

| Шаг | Что делает | При сбое |
|---|---|---|
| `detect` | Выводит существующие GPG-ключи | Если ключи есть, мастер предлагает использовать один; если нет — переходит к `generate` |
| `generate` | Создаёт новый ключ (или переиспользует существующий) | Retry / skip / abort |
| `export` | Экспортирует публичный ключ в `gpg-public-key.asc`, перехватывает приватный ключ в память | Retry / skip / abort |
| `git-config` | Выставляет шесть ключей подписи git в локальном репозитории | Retry / skip / abort |
| `github` | Загружает публичный ключ, выставляет секреты репозитория, открывает PR | Retry / skip / abort |
| `publish` | Публикует публичный ключ на сервере(ах) ключей | Retry / skip / abort |

Пропуск шага записывает его как завершённый в файле состояния, поэтому мастер возобновляется со следующего шага при повторном запуске.
