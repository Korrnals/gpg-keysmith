# `generate`

Сгенерировать новый GPG-ключ, управляя `gpg --gen-key` через batch-файл параметров.

## Синопсис

```
keysmith generate [флаги]
```

`generate` пишет batch-файл параметров и запускает `gpg --gen-key --batch --pinentry-mode loopback --passphrase-fd 0`, подавая парольную фразу через stdin. Парольная фраза собирается через маскированный промпт `survey.Password` — она никогда не появляется в batch-файле, аргументах процесса и логах.

Используйте `--name` и `--email`, чтобы пропустить интерактивные промпты для этих полей (неинтерактивный режим). Парольная фраза **всегда** запрашивается через маскированное поле, даже в неинтерактивном режиме.

## Флаги

| Флаг | Тип | По умолчанию | Описание |
|---|---|---|---|
| `--comment` | string | `""` | Комментарий для user id ключа (необязательно) |
| `--email` | string | `""` | Email для user id ключа |
| `--expiry` | string | `"0"` | Спецификация срока действия (`0` = бессрочно, `2y` = 2 года) |
| `--key-length` | int | `4096` | Длина RSA-ключа в битах |
| `--name` | string | `""` | Настоящее имя для user id ключа |
| `-h`, `--help` | bool | `false` | Вывести помощь для `generate` |
| `--config` | string | `~/.config/gpg-keysmith/config.yaml` | Путь к файлу конфигурации (глобальный флаг) |

## Примеры

### Полностью интерактивно (промпт для всего, кроме ввода парольной фразы)

```bash
keysmith generate
```

Survey запрашивает имя, email, комментарий, срок действия и длину ключа; парольная фраза собирается через маскированное поле.

### Неинтерактивно с явными именем и email

```bash
keysmith generate --name "Jane Doe" --email jane@example.com
```

Запрашивается только парольная фраза (маскированно); остальное берётся из флагов.

### Сгенерировать ключ на 2 года с комментарием для идентификации проекта

```bash
keysmith generate --name "Jane Doe" --email jane@example.com \
  --comment "acme-corp signing" --expiry 2y
```

### Сгенерировать 3072-битный ключ (меньше, быстрее)

```bash
keysmith generate --name "Jane Doe" --email jane@example.com --key-length 3072
```

### Использовать значения по умолчанию из конфига

```bash
keysmith config init   # пишет шаблон со значениями: RSA 4096, expire "0"
keysmith generate --name "Jane Doe" --email jane@example.com
```

Значения `key.length` и `key.expire` из конфига используются, если не перекрыты флагами.

## Замечания

- **Обработка парольной фразы.** Парольная фраза подаётся в `gpg` через `--passphrase-fd 0` (stdin) с `--pinentry-mode loopback`. Она никогда не появляется в batch-файле, не передаётся как аргумент CLI и не логируется. См. [Безопасность](../security.md).
- **Алгоритм.** Поддерживается только RSA (`key.type: RSA` в конфиге; `--key-length` задаёт длину в битах). ECC/EdDSA вне области v1.0.0.
- **`%no-protection` запрещён.** Batch-файл намеренно опускает директиву `%no-protection` — `gpg-keysmith` всегда требует парольную фразу.
- **Runtime-зависимость.** Требуется бинарник `gpg` (GnuPG 2.x) в `PATH`.
- **Перекрытие конфига.** `--key-length` и `--expiry` перекрывают `config.key.length` и `config.key.expire`; `--name` / `--email` / `--comment` в конфиге не хранятся (это идентичность конкретного ключа, а не значения по умолчанию).
